ДЛЯ ПЕРЕНОСА САЙТА НА CMS БИТРИКС НА ДРУГОЙ ХОСТИНГ, НУЖНО:

1) создать резервную копию сайта Админка -> Настройки -> Инструменты -> Резервное копирование -> создать резеврную копию ->

   в форме пункт Резервное копирование -> в папке сайта -> пункт Параметры -> включить экспертные настройки ->
 
   длительность шага 14, интервал 3 сек., максимальный размер несжатых данных 2000;

2) в Настройках -> Резервное копирование -> Список резервных копий -> нажать на бургер -> ссылка для переноса;

3) зайти на хостинг, если нужно создать сайт с базой данных -> зайти в файловую систему сайта -> папка Корень сайта ->

   загрузить туда файл restore.php -> войти на домен и прописать в url с права от домена "/restore.php";

4) снять пароль с папки сайта с резервной копией через хостинг, если он есть -> вставить ссылку на резеврную копию;

5) как только произойдет распаковка архива и вылезет ошибка, то нужно переименовать файл .htaccess.restore в

   .htaccess;

6) в файле, расположенному по пути: /bitrix/php_interface/dbconn.php закомментировать строку "define("BX_UTF", true);";

7) когда передем к пункту с восстановлением базы данных, нужно будет ввести правильный сервер базы данных 

   (он должен быть, скорее всего, не localhost (на timeweb localhost)), затем имя БД, логин и пароль от БД на сайте с резервной копией;

8) удалить через диалоговое окно на странице домен/restore.php локальную копию и служебные скрипты;

9) если не заработает сайт, то нужно изменить версию php на более раннюю в настройках сайта.
